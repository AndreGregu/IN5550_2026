2: Evaluation 2_draft:
	Improvements:
		Added metadata features in addition to the text
		Original model only used "text" as a feature with TF-IDF → Ridge regression
		From this i addded some extra structures such as: 
			- text_length
			- word_count
			- is_words
			- collection_phase
		the model then uses TF-IDF(text) + metadata → Ridge regression
		text_length and word_count helps because the strucutre of essays are different from short-emotion words. 
		is_word tells the model whether the entry is a full essay or a feeling-word-list which matters because those two text types are explicitly different in the task. 
		collection_phase provides some weak temporal/contextual information

	Performance: 	
		Evaluation on Valence:
		r_between 0.734 (1.31e-16)

		Evaluation on arousal:

	Key differences: 
		valence:
		r_composite: 0.597 → 0.612
		r_between: 0.714 → 0.734
		r_within: 0.448 → 0.453 (barely noticable)
		
		arousal: 
		r_between: 0.564 → 0.542
		r_within:  0.311 → 0.313
		r_comp:    0.447 → 0.435
			
	
	Interpretation: 
		Better capture global patterns (between users)
		But still not much better at within-user variation over time
		The added features does not help model's ability to predict arousal
			Proably because metadata adds noise
			Propperly formulated analysis: 
			unlike valence, arousal does not benefit from simple structural features such as text length or collection phase. This sugests that arousal is more dependent on nuanced semantic and expressive cues rather than surface-level properties.
	Bottleneck: 
		modeling user-specific dynamics
